## ✨ المميزات

- **وكيل MNC Architect AI** — مهندس استشاري سعودي يولّد تقريراً معمارياً كاملاً (16 قسماً) من بيانات الأرض.
- **مساحة عمل لكل مشروع** بها 13 أداة ذكاء اصطناعي.
- **حاسبات هندسية حقيقية تعمل فوراً** (بدون أي مفاتيح): التكلفة، العائد ROI، المواقف — بأسعار السعودية التقريبية.
- **مساعد محادثة معماري** داخل كل مشروع: «قلّل التكلفة»، «زد الشقق»، «غيّر الطراز».
- **مركز تصدير** كامل: PDF · Word · Excel · JSON — جميعها بهوية MNC.
- **نظام صلاحيات RBAC** بـ8 أدوار (يتحقق في الواجهة وفي الـ API).
- **لوحة تحكم** بإحصائيات ورسوم بيانية.
- **ثنائي اللغة** عربي (RTL) / إنجليزي (LTR) عبر نظام ترجمة مركزي.
- **داكن/فاتح** بنفس هوية MNC (Navy & Gold).

> 💡 **وضع تجريبي (Demo Mode):** المنصة تعمل بالكامل **بدون أي مفاتيح**. سجّل الدخول بأي بريد، وستجد مشاريع جاهزة وتقارير تُولّد بالمعادلات الهندسية فوراً.

## 🚀 التشغيل السريع

```bash

npm run dev
```

## 🔐 التفعيل الكامل (اختياري)

انسخ ملف البيئة واضبط المتغيرات:

```bash
cp .env.example .env.local
```

| المتغيّر | الغرض |
|---|---|
| `ANTHROPIC_API_KEY` | تفعيل تحليل المهندس الاستشاري الكامل (بدلاً من التقرير التجريبي) |
| `ANTHROPIC_MODEL` | الموديل (افتراضي `claude-sonnet-4-6`) |
| `NEXT_PUBLIC_FIREBASE_*` | تفعيل المصادقة والحفظ السحابي على Firestore |
| `FIREBASE_ADMIN_*` | عمليات الخادم الآمنة (اختياري) |

بدون `ANTHROPIC_API_KEY` → تقارير تجريبية بالمعادلات.
بدون متغيرات Firebase → الحفظ محلياً في المتصفح (localStorage).

│   └── layout.tsx               # الخطوط + جميع الـ Providers

│   ├── ui/                      # عناصر واجهة (Button, Card, Badge, …)
│   ├── layout/                  # Sidebar, Topbar, Logo
│   ├── dashboard/               # بطاقات الإحصائيات + الرسوم
│   └── workspace/               # سجل الأدوات + عارض التقرير + المساعد + الحاسبات
├── lib/
│   ├── ai/agent.ts              # شخصية وكيل MNC + مخطط الإخراج JSON
│   ├── calc/engineering.ts      # المعادلات الهندسية + باني التقرير التجريبي
│   ├── domain/types.ts          # الأنواع (Project, ArchitectReport, …)
│   ├── rbac/permissions.ts      # نظام الصلاحيات (8 أدوار)
│   ├── data/projects-store.ts   # Firestore ← fallback localStorage
│   ├── export/report-export.ts  # PDF · Word · Excel · JSON
│   ├── firebase/client.ts       # تهيئة Firebase (مع علم التفعيل)
│   └── i18n/                     # الترجمة المركزية (ar / en)
└── providers/                   # Auth, Theme, Query, I18n

## 🧩 نقاط ربط الذكاء الاصطناعي (لتطوير الأدوات لاحقاً)

الأدوات الأربع الأساسية تعمل فوراً (**التقرير** + **حاسبات** التكلفة/العائد/المواقف).
بقية أدوات الـ13 مجهّزة بنقطة ربط موحّدة (`// AI HOOK`) عبر مسار `/api/architect`،
بحيث يمكن توصيل كل أداة بنفس البنية دون إعادة هيكلة. راجع:

- `src/app/api/architect/route.ts` — استدعاء Claude + تحليل JSON + الرجوع الآمن.
- `src/lib/ai/agent.ts` — شخصية الوكيل ومخطط الإخراج.
- `src/components/workspace/tools-registry.ts` — سجل الأدوات (kind / ready).

## ☁️ النشر على Vercel

1. ارفع المشروع إلى مستودع Git.
2. استورده في Vercel.
3. أضِف متغيرات البيئة (`ANTHROPIC_API_KEY`, `NEXT_PUBLIC_FIREBASE_*`).
4. عند تغيير متغيرات البيئة: احذف الكاش وأعد النشر (`Redeploy`).

```bash
npm run build   # بناء الإنتاج
npm run start   # تشغيل الإنتاج محلياً
```
